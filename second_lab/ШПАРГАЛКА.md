## 📦 ПАКЕТЫ (PACKAGES)

### Что это?
**Пакет** = папка для группировки связанных классов.

### Зачем?
- Организация кода (не всё в одной куче)
- Избежание конфликтов имён
- Логическая структура проекта

### Как использовать?

**1. Объявление пакета (ПЕРВАЯ строка в файле):**
```java
package matrix;  // этот класс в пакете matrix
```

**2. Импорт классов из других пакетов:**
```java
import exceptions.MatrixException;  // импортируем класс из пакета exceptions
```

### Важно!
- `package` - ПЕРВАЯ строка в файле (до всего остального)
- Название пакета = название папки
- `src/matrix/Matrix.java` → package должен быть `matrix`, а НЕ `fird_lab.src.matrix`
- Папка `src/` - это корень, от неё начинаются пакеты

## 🧬 НАСЛЕДОВАНИЕ (EXTENDS)

### Что это?
**Наследование** - когда один класс получает все поля и методы другого класса.

```java
public class SquareMatrix extends Matrix {
    // SquareMatrix НАСЛЕДУЕТ всё от Matrix:
    // - поля: data, rows, columns
    // - методы: sum, product, getElement, setElement, toString, equals
}
```

### Терминология:
- **Matrix** - родительский класс (parent, superclass, базовый класс)
- **SquareMatrix** - дочерний класс (child, subclass, наследник)

### Зачем?
- Переиспользование кода (не писать одно и то же)
- Расширение функционала
- Специализация (SquareMatrix - специальный случай Matrix)

### Что наследуется?
✅ Все `public` и `protected` методы
✅ Все поля (но `private` поля недоступны напрямую!)
❌ Конструкторы НЕ наследуются!

### Доступ к приватным полям родителя:

**НЕЛЬЗЯ:**
```java
public class SquareMatrix extends Matrix {
    public void test() {
        this.rows = 5;  // ❌ ОШИБКА! rows - private в Matrix
    }
}
```

**МОЖНО через геттеры:**
```java
public class SquareMatrix extends Matrix {
    public void test() {
        int size = this.getRows();  // ✅ Работает!
    }
}
```

## 🔨 КОНСТРУКТОРЫ И super()

### Конструкторы НЕ наследуются!

Если в родителе есть конструктор:
```java
public class Matrix {
    public Matrix(int rows, int columns) { ... }
}
```

То в наследнике НУЖНО вызвать конструктор родителя через `super()`:

```java
public class SquareMatrix extends Matrix {
    public SquareMatrix(int size) {
        super(size, size);  // Вызываем конструктор Matrix

        // Дальше свой код...
        for (int i = 0; i < size; i++) {
            this.setElement(i, i, 1);
        }
    }
}
```

### ПРАВИЛО: super() ВСЕГДА ПЕРВЫМ!

**ПРАВИЛЬНО:**
```java
public SquareMatrix(int size) {
    super(size, size);  // ✅ ПЕРВАЯ строка
    // остальной код...
}
```

**НЕПРАВИЛЬНО:**
```java
public SquareMatrix(int size) {
    int x = 5;          // ❌ ОШИБКА!
    super(size, size);  // super() должен быть первым!
}
```

### Что такое super()?

`super()` - вызов конструктора РОДИТЕЛЬСКОГО класса.

**Аналогия:**
Ты строишь дом (SquareMatrix). Но сначала нужно построить ФУНДАМЕНТ (Matrix) через `super()`, а потом уже добавлять свои элементы.

## 🔄 ПЕРЕОПРЕДЕЛЕНИЕ МЕТОДОВ (@Override)

### Что это?

**Переопределение** - изменение поведения метода родителя в наследнике.

### Пример:

**В Matrix:**
```java
public Matrix sum(Matrix other) {
    // возвращает Matrix
    return new Matrix(...);
}
```

**В SquareMatrix:**
```java
@Override  // говорим: "я переопределяю метод родителя"
public SquareMatrix sum(SquareMatrix other) {
    // возвращает SquareMatrix (более специфичный тип)
    return new SquareMatrix(...);
}
```

### Зачем @Override?

- Компилятор проверяет, что метод действительно переопределяется
- Если ошибёшься в названии - компилятор предупредит
- Документирует код (понятно, что это переопределение)

### Правило сигнатуры:

Чтобы переопределить метод, нужно:
- ✅ То же имя метода
- ✅ Те же параметры (тип и порядок)
- ✅ Совместимый возвращаемый тип (можно более специфичный)

## ⚠️ ИСКЛЮЧЕНИЯ (EXCEPTIONS)

### Что это?

**Исключение** - способ сообщить программе об ошибке.

### Создание своего класса исключения:

```java
package exceptions;

public final class MatrixException extends RuntimeException {
    public MatrixException(String message) {
        super(message);  // передаём сообщение родителю
    }
}
```

### Почему extends RuntimeException?

- `RuntimeException` - встроенный класс Java для ошибок
- У него уже есть всё нужное (хранение сообщения, вывод и т.д.)
- Мы просто добавляем своё имя класса

### Как бросать исключения:

```java
if (rows <= 0 || columns <= 0) {
    throw new MatrixException("Невозможно создать матрицу с размером " + rows + " x " + columns);
}
```

### Как ловить исключения:

```java
try {
    Matrix m = new Matrix(0, 5);  // вызовет исключение
} catch (MatrixException e) {
    System.out.println("Ошибка: " + e.getMessage());
}
```

### Когда бросать исключения в нашей лабе:

- ❌ Неверный размер матрицы (≤ 0)
- ❌ Сложение матриц разных размеров
- ❌ Умножение несовместимых матриц (columns1 ≠ rows2)
- ❌ Выход за границы массива (индексы < 0 или ≥ размера)

## 🔒 МОДИФИКАТОР final

### Что это?

`final` = "нельзя изменить"

### Где используется:

#### 1. final ПЕРЕМЕННАЯ (константа)

```java
private final int rows;  // Нельзя изменить после присвоения
```

**Пример:**
```java
public Matrix(int rows, int columns) {
    this.rows = rows;     // ✅ Первое присвоение - OK
    this.rows = 10;       // ❌ ОШИБКА! Уже присвоено!
}
```

**Зачем:** Размер матрицы не меняется после создания.

#### 2. final МЕТОД

```java
public final String toString() { ... }
```

**Что значит:** Наследники НЕ могут переопределить этот метод.

**Зачем:** Логика вывода матрицы одинаковая для всех, переопределять не нужно.

#### 3. final КЛАСС

```java
public final class MatrixException extends RuntimeException { ... }
```

**Что значит:** От этого класса НЕЛЬЗЯ наследоваться.

**Зачем:** От исключения не планируем наследоваться.

#### 4. final НЕЛЬЗЯ на конструкторах!

```java
public final Matrix(int rows, int columns) { ... }  // ❌ ОШИБКА КОМПИЛЯЦИИ!
```

**Почему:** Конструкторы не наследуются, поэтому `final` бессмысленен.

## ⚖️ МЕТОД equals()

### Зачем нужен?

Сравнивать объекты по **содержимому**, а не по ссылке.

```java
Matrix m1 = new Matrix(2, 2);
Matrix m2 = new Matrix(2, 2);

m1 == m2;        // false (разные объекты в памяти)
m1.equals(m2);   // true (одинаковое содержимое)
```

### Правильная реализация:

```java
@Override
public boolean equals(Object obj) {
    // 1. Сравниваем ссылки (оптимизация)
    if (this == obj) {
        return true;
    }

    // 2. Проверка на null и тип
    if (obj == null || getClass() != obj.getClass()) {
        return false;
    }

    // 3. Приводим к типу Matrix
    Matrix other = (Matrix) obj;

    // 4. Сравниваем размеры
    if (this.rows != other.rows || this.columns != other.columns) {
        return false;
    }

    // 5. Сравниваем элементы
    for (int i = 0; i < this.rows; i++) {
        for (int j = 0; j < this.columns; j++) {
            if (this.data[i][j] != other.data[i][j]) {
                return false;
            }
        }
    }

    return true;
}
```

### Почему сигнатура equals(Object obj), а не equals(Matrix other)?

Потому что мы **переопределяем** метод из класса `Object`:

```java
// В классе Object:
public boolean equals(Object obj) { ... }
```

Чтобы переопределить, нужна **точно такая же сигнатура**: `equals(Object obj)`.

Если написать `equals(Matrix other)`, это будет **новый метод** (перегрузка), а не переопределение!

## 🎯 КЛАСС Object И getClass()

### Класс Object - родитель ВСЕХ классов

В Java **ВСЕ** классы автоматически наследуются от `Object`:

```java
public class Matrix {  // на самом деле: public class Matrix extends Object
    ...
}
```

### Методы из Object (доступны везде):

- `getClass()` - возвращает класс объекта
- `equals(Object obj)` - сравнение объектов
- `toString()` - преобразование в строку
- `hashCode()` - хеш-код объекта

### Метод getClass()

Возвращает объект типа `Class`, который описывает класс объекта.

```java
Matrix m = new Matrix(2, 2);
SquareMatrix sm = new SquareMatrix(3);

m.getClass();   // вернёт: class matrix.Matrix
sm.getClass();  // вернёт: class matrix.SquareMatrix
```

### Использование в equals():

```java
if (getClass() != obj.getClass()) {
    return false;  // Разные классы - не равны
}
```

**Это сравнение ссылок!** Для каждого класса существует **один** объект `Class`, поэтому можно сравнивать через `!=`.

**Пример:**
```java
Matrix m = new Matrix(2, 2);
SquareMatrix sm = new SquareMatrix(2);

m.getClass() != sm.getClass()  // true (Matrix ≠ SquareMatrix)
```

## 🔄 ПРИВЕДЕНИЕ ТИПОВ (CASTING)

### Проблема:

В методе `equals(Object obj)` параметр имеет тип `Object`.

```java
public boolean equals(Object obj) {
    obj.getRows();  // ❌ ОШИБКА! У Object нет метода getRows()
}
```

### Решение: приведение типа

```java
Matrix other = (Matrix) obj;  // "преобразуй obj к типу Matrix"
other.getRows();  // ✅ Теперь работает!
```

### Синтаксис:

```java
НовыйТип переменная = (НовыйТип) старая_переменная;
```

### Что происходит?

Объект **НЕ меняется**! Меняется только то, как **компилятор на него смотрит**.

**Аналогия:**

```
Коробка с телефоном внутри
│
├─ obj (тип Object) → "это просто вещь"
│  можно: взять, положить
│  нельзя: звонить, писать SMS
│
└─ other (тип Matrix) → "это телефон!"
   можно: звонить, писать SMS, getRows(), getElement()...
```

### ОПАСНОСТЬ: ClassCastException

Что будет, если приведёшь к неправильному типу?

```java
Object obj = "строка";
Matrix m = (Matrix) obj;  // ❌ ОШИБКА В RUNTIME! ClassCastException
```

### Поэтому ВСЕГДА проверяем ПЕРЕД приведением:

```java
if (obj == null || getClass() != obj.getClass()) {
    return false;  // если obj не того типа - выходим
}

// Сюда попадаем ТОЛЬКО если obj точно Matrix!
Matrix other = (Matrix) obj;  // ✅ Безопасно!
```

## 🔑 ГЕТТЕРЫ И СЕТТЕРЫ

### Зачем нужны?

Доступ к `private` полям извне класса.

### Private поля:

```java
private int rows;      // Никто снаружи не может обратиться
private int columns;
```

### Геттеры (получить значение):

```java
public int getRows() {
    return this.rows;
}

public int getColumns() {
    return this.columns;
}
```

### Сеттеры (установить значение):

```java
public void setElement(int row, int column, int value) {
    // Можно добавить проверки!
    if (row < 0 || row >= this.rows) {
        throw new MatrixException("Индекс вне границ");
    }
    this.data[row][column] = value;
}
```

### Зачем делать поля private?

**Инкапсуляция** - скрытие внутреннего устройства класса.

**Плюсы:**
- Контроль доступа (можно добавить проверки в сеттере)
- Можно изменить внутреннюю реализацию без изменения интерфейса
- Защита от случайных изменений

**Пример:**

```java
// ПЛОХО:
public int rows;  // Кто угодно может написать: matrix.rows = -100; ❌

// ХОРОШО:
private int rows;
public void setRows(int rows) {
    if (rows <= 0) throw new MatrixException("Размер должен быть > 0");
    this.rows = rows;
}
```

## 📝 АННОТАЦИЯ @Override

### Что это?

`@Override` - метка компилятору: "я переопределяю метод родителя".

```java
@Override
public boolean equals(Object obj) { ... }
```

### Зачем?

1. **Проверка компилятором:**
   ```java
   @Override
   public boolean equal(Object obj) { ... }  // ❌ ОШИБКА! Опечатка: equal вместо equals
   ```
   Компилятор скажет: "Ты написал @Override, но такого метода в родителе нет!"

2. **Документирование кода:**
   Когда читаешь код, сразу видно, что метод переопределён.

3. **Защита от изменений:**
   Если родитель поменяет сигнатуру метода, компилятор предупредит.

### Где использовать:

- ✅ `equals(Object obj)` - переопределяем из Object
- ✅ `toString()` - переопределяем из Object
- ✅ `sum()` в SquareMatrix - переопределяем из Matrix

## 🧮 ОСОБЕННОСТИ МАТРИЦ

### Matrix vs SquareMatrix

| Matrix | SquareMatrix |
|--------|-------------|
| Прямоугольная (rows × columns) | Квадратная (size × size) |
| Конструктор: Matrix(rows, columns) | Конструктор: SquareMatrix(size) |
| Создаёт нулевую матрицу | Создаёт единичную матрицу |
| sum() возвращает Matrix | sum() возвращает SquareMatrix |

### Единичная матрица:

```
1 0 0
0 1 0
0 0 1
```

На диагонали (где i == j) стоят единицы, остальное - нули.

```java
for (int i = 0; i < size; i++) {
    for (int j = 0; j < size; j++) {
        if (i == j) {
            this.setElement(i, j, 1);  // диагональ
        } else {
            this.setElement(i, j, 0);  // остальное
        }
    }
}
```

### Сложение матриц:

Можно складывать только матрицы **ОДИНАКОВОГО размера**!

```
[1 2]   [5 6]   [6  8]
[3 4] + [7 8] = [10 12]
```

```java
if (this.rows != other.rows || this.columns != other.columns) {
    throw new MatrixException("Размеры не совпадают!");
}
```

### Умножение матриц:

Можно умножать только если: **columns первой == rows второй**

```
[2×3] × [3×2] = [2×2]  ✅ OK (3 == 3)
[2×3] × [2×3] = ???    ❌ ОШИБКА (3 ≠ 2)
```

```java
if (this.columns != other.rows) {
    throw new MatrixException("Несовместимые размеры!");
}

Matrix result = new Matrix(this.rows, other.columns);
```

## 🎓 ВОЗМОЖНЫЕ ВОПРОСЫ ПРЕПОДАВАТЕЛЯ

### 1. "Что такое наследование?"

**Ответ:** Наследование - это механизм ООП, когда один класс (наследник) получает все поля и методы другого класса (родителя). В нашей лабе SquareMatrix наследуется от Matrix через `extends`, получая все его методы и поля.

### 2. "Зачем нужен super()?"

**Ответ:** `super()` вызывает конструктор родительского класса. Это нужно, чтобы инициализировать поля родителя. Конструкторы не наследуются, поэтому нужно явно вызывать родительский конструктор. `super()` должен быть первой строкой в конструкторе наследника.

### 3. "Почему equals принимает Object, а не Matrix?"

**Ответ:** Потому что мы переопределяем стандартный метод `equals(Object obj)` из класса Object. Чтобы переопределить метод, сигнатура должна совпадать. Если написать `equals(Matrix other)`, это будет перегрузка (новый метод), а не переопределение.

### 4. "Что делает getClass()?"

**Ответ:** `getClass()` - метод из класса Object, который возвращает объект типа Class, описывающий класс объекта. Используется для проверки точного типа объекта в runtime. В equals() мы проверяем `getClass() != obj.getClass()`, чтобы убедиться, что объекты одного класса.

### 5. "Зачем приведение типов (Matrix) obj?"

**Ответ:** Параметр `obj` имеет тип Object, у которого нет методов Matrix (getRows, data и т.д.). Приведение типа говорит компилятору: "я знаю, что это Matrix, позволь обращаться к его методам". Перед приведением мы проверяем тип через getClass(), чтобы избежать ClassCastException.

### 6. "Почему поля private, а не public?"

**Ответ:** Это инкапсуляция - принцип ООП. Private поля скрывают внутреннюю реализацию класса. Доступ к ним предоставляется через геттеры/сеттеры, где можно добавить проверки. Например, в setElement мы проверяем границы массива перед изменением. Это защищает от некорректных данных.

### 7. "Зачем нужен final?"

**Ответ:**
- `final` для полей (rows, columns) - значение нельзя изменить после инициализации. Размер матрицы не должен меняться после создания.
- `final` для методов (toString) - метод нельзя переопределить в наследниках. Логика вывода одинаковая для всех.
- `final` для классов (MatrixException) - от класса нельзя наследоваться.

### 8. "Почему в SquareMatrix переопределили sum()?"

**Ответ:** Чтобы метод возвращал SquareMatrix вместо Matrix. Это более точный тип - сумма двух квадратных матриц тоже квадратная. Также это требование задания - переопределить sum в SquareMatrix.

### 9. "Когда бросаются исключения?"

**Ответ:**
- Неверный размер при создании (≤ 0)
- Сложение матриц разных размеров
- Умножение несовместимых матриц (columns первой ≠ rows второй)
- Выход за границы массива при get/setElement

### 10. "Чем Matrix отличается от SquareMatrix?"

**Ответ:**
- Matrix - прямоугольная (rows × columns), SquareMatrix - квадратная (size × size)
- Matrix создаёт нулевую матрицу, SquareMatrix - единичную (1 на диагонали)
- SquareMatrix наследуется от Matrix и переопределяет sum()

## 🔥 КЛЮЧЕВЫЕ ТЕРМИНЫ НА АНГЛИЙСКОМ

- **Inheritance** - наследование
- **Superclass / Parent class** - родительский класс
- **Subclass / Child class** - дочерний класс (наследник)
- **Override** - переопределение метода
- **Overload** - перегрузка метода (другие параметры)
- **Encapsulation** - инкапсуляция
- **Exception** - исключение
- **Type casting** - приведение типов
- **Getter** - метод получения значения
- **Setter** - метод установки значения
- **Package** - пакет
- **Import** - импорт
- **Final** - финальный (неизменяемый)
